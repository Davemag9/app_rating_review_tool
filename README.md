## Локальний запуск

### Вимоги

- Python 3.11+
- Ключ Gemini API ([отримати безкоштовно](https://aistudio.google.com/app/apikey))

### 1. Клонувати репозиторій

```bash
git clone <repo-url>
cd app_rating_review_tool
```

### 2. Створити та активувати віртуальне середовище

```bash
python -m venv venv
source venv/bin/activate        # macOS / Linux
# або
venv\Scripts\activate           # Windows
```

### 3. Встановити залежності

```bash
pip install -r requirements.txt
```

### 4. Налаштувати змінні середовища

Створіть файл `.env` у корені проєкту:

```dotenv
GEMINI_API_KEY=ваш_ключ_тут
```

> Без цього ключа ендпоінти аналізу (`/analyze`) повертатимуть `503`.

### 5. Запустити сервер

```bash
uvicorn api.main:app --reload
```

Після запуску доступно:


| URL                              | Опис                                 |
| -------------------------------- | ------------------------------------ |
| `http://localhost:8000/`         | Редирект на Gradio-інтерфейс         |
| `http://localhost:8000/frontend` | Веб-інтерфейс Gradio                 |
| `http://localhost:8000/docs`     | Інтерактивна документація Swagger UI |
| `http://localhost:8000/redoc`    | ReDoc документація                   |
| `http://localhost:8000/health`   | Перевірка стану сервісу              |

## Архітектура та ключові рішення

### Стек

- **FastAPI** — основний веб-фреймворк: автоматична документація, валідація через Pydantic, async-підтримка.
- **Gradio** — вбудований веб-інтерфейс, змонтований на `/frontend` всередині того самого FastAPI-процесу.
- **google-play-scraper** — збирає відгуки без офіційного API.
- **Google Gemini** (`google-genai`) — класифікація тональності та генерація рекомендацій.
- **scikit-learn / NLTK** — TF-IDF уніграми, біграми та тріграми для NLP-аналізу без LLM.

### In-memory сховище без бази даних

Усі дані тримаються у `AppStore` — потоко-безпечному `OrderedDict` з LRU-витісненням (максимум 3 застосунки одночасно). Це спрощує деплой: жодних міграцій, жодних зовнішніх сервісів. Підходить для поодинокого використання; при потребі масштабування сховище замінюється на Redis або PostgreSQL без зміни бізнес-логіки.

### Дворівневий пайплайн аналізу

Збір відгуків і AI-аналіз навмисно розділені на два окремих запити:

1. `**/collect`** — швидко, не потребує Gemini API, повертає метрики.
2. `**/analyze**` — повільніше, робить кілька звернень до Gemini, може коштувати токени.

# !! Для того щоб зробити аналіз, необхідно спершу запустити отримання даних !!

Такий поділ дає змогу переглядати метрики без витрат на LLM та повторно запускати аналіз без повторного скрапінгу.

### Гібридне вилучення ключових слів

Негативні ключові слова вилучаються двома способами:

- **LLM (Gemini)** — виявляє семантичні теми, навіть якщо вони сформульовані по-різному.
- **NLP (TF-IDF + n-грами)** — статистично значущі терміни, не залежить від LLM та його галюцинацій.

Обидва результати об'єднуються при генерації рекомендацій для більшої повноти.

### Стратегія вибірки

Завантажується пул із `pool_size` найновіших відгуків, з якого випадково береться `sample` штук. Це зменшує зміщення на користь найновіших записів і водночас обмежує кількість запитів до скрапера.

### Єдиний процес для спрощення деплою

Gradio-інтерфейс і REST API запускаються в одному uvicorn-процесі. `Procfile` містить єдину команду — зручно для Railway, Render та подібних платформ.
